## 环境配置

### 1. 修改配置文件

#### project.config.json
```json
{
    "appid": "你的AppID",
    "projectname": "萌宠大冒险"
}
```

#### GameConfig.ts
```typescript
WECHAT: {
    APP_ID: '你的AppID',
    ENV_ID: '你的云开发环境ID'
}
```

### 2. 配置云开发

#### 数据库集合创建
```javascript
// 在云开发控制台创建以下集合
1. users - 用户数据
2. leaderboard - 排行榜
3. items - 道具数据
```

#### 云函数部署
```bash
# 部署登录云函数
cd server/cloudfunctions/login
npm install
wx cloud functions deploy --name login

# 部署排行榜云函数
cd ../updateLeaderboard
npm install
wx cloud functions deploy --name updateLeaderboard

# 部署获取排行榜云函数
cd ../getLeaderboard
npm install
wx cloud functions deploy --name getLeaderboard
```

#### 数据库权限设置
```json
// users 集合权限
{
    "read": "doc._openid == auth.openid",
    "write": "doc._openid == auth.openid"
}

// leaderboard 集合权限
{
    "read": true,
    "write": "doc._openid == auth.openid"
}
```

### 3. 广告配置

#### 申请广告位
1. 登录微信公众平台

3. 申请开通流量主

#### 配置广告ID

## 构建发布

### 1. Cocos Creator构建

#### 构建配置
```
平台: 微信小游戏
模板: default
Start Scene: MainScene
参与构建的场景:
  - MainScene
  - BattleScene
  - GachaScene
  - BagScene

构建路径: ./build/wechatgame
```

#### 构建步骤
1. 打开Cocos Creator
2. 选择"项目" -> "构建发布"
3. 选择"微信小游戏"
4. 配置AppID
5. 点击"构建"

### 2. 微信开发者工具

#### 导入项目
1. 打开微信开发者工具
2. 选择"导入项目"
3. 选择构建目录 `build/wechatgame`
4. 输入AppID
5. 点击"导入"

#### 上传代码
1. 点击右上角"上传"
2. 填写版本号（如1.0.0）
3. 填写项目备注
4. 点击"上传"

## 提交审核

### 1. 准备材料

#### 基本信息
- [ ] 游戏名称
- [ ] 游戏简介
- [ ] 游戏图标（正方形，不小于512x512）
- [ ] 游戏截图（5张，不小于480x800）

#### 资质材料
- [ ] 软件著作权证书（如有）
- [ ] ICP备案（如有）
- [ ] 文网文（如有）

#### 合规材料
- [ ] 隐私政策文档
- [ ] 用户协议文档
- [ ] 适龄提示（12+）

### 2. 填写审核信息

#### 功能页面
```
主页面路径: pages/index/index
功能描述: 萌宠收集养成游戏
```

#### 隐私设置
```
收集用户信息: 是
收集内容: 用户昵称、头像
用途: 游戏内显示、排行榜
```

### 3. 提交审核
1. 登录微信公众平台
2. 进入"版本管理"
3. 找到开发版本
4. 点击"提交审核"
5. 填写审核信息
6. 等待审核结果（通常1-3个工作日）

## 上线发布

### 1. 审核通过后
1. 收到审核通过通知
2. 登录微信公众平台
3. 进入"版本管理"

## 常见问题

### Q: 审核被拒常见原因？
A: 
1. 游戏内容违规
2. 缺少隐私政策
3. 用户体验差（卡顿、闪退）
4. 诱导分享
5. 版权问题

### Q: 如何提高审核通过率？
A:
1. 确保游戏内容健康
2. 完善隐私政策和用户协议
3. 测试充分，无明显Bug
4. 提供清晰的审核说明
5. 准备完整的资质材料

### Q: 上线后如何更新？
A:
1. 小更新：使用热更新
2. 大更新：提交新版本审核
3. 紧急修复：可以撤回版本重新提交
